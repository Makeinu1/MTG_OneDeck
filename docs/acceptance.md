# G1/G2 受け入れシナリオ(E2E)

実行者: レビュー担当(メインセッション)がブラウザ実機で実行する。
**判定規則: 1ステップでも期待結果と異なれば不合格。修正後はステップ1から全再実行**(部分再試験は不可)。
全ステップを通じて **ブラウザコンソールにエラーが1件も出ないこと**(warning は記録のみ)。

操作は原則**右クリックメニュー**(全操作の代替手段)で行う。D&D は最終ステップで抜き取り確認。
M4.5以降、シングル左クリックはメニューを開かない(右クリック=メニュー、ダブルクリック=クイック操作)。

## 前提
- `docs/fixtures/test-deck.txt`: 統率者1体を含む100枚のEDHデッキリスト。日本語カード名を30行以上含み、両面カード(transform または modal_dfc)を1枚以上、トークンを生成するカードを1枚以上含むこと(G1実行前にレビュー担当が作成し、Scryfallで全カード名の実在を確認する)
- `npm run dev` でアプリ起動(G2では `npx serve dist`)
